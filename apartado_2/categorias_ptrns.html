Categorias de Patrones de Diseño

Los patrones de diseño se dividen en tres categorías principales, según el tipo de problema que resuelven:

  • Patrones Creacionales: Se centran en la forma de crear objetos. Ayudan a controlar y hacer más flexible el proceso de instanciar clases, para que no dependas siempre del mismo método rígido. Ejemplos comunes son el Singleton (que garantiza una sola instancia de una clase), el Factory Method (que decide qué tipo de objeto crear según la situación) o el Abstract Factory (que crea familias completas de objetos relacionados).
  • Patrones Estructurales: Se ocupan de cómo se componen y organizan las clases y objetos para formar estructuras más grandes y útiles. Sirven para que partes del código que no encajan naturalmente puedan trabajar juntas sin problemas. Algunos ejemplos son el Adapter (que hace compatibles clases incompatibles), el Decorator (que agrega funcionalidades extras sin modificar la clase original) o el Proxy (que actúa como intermediario para controlar el acceso a un objeto).
  • Patrones de Comportamiento: Definen cómo interactúan los objetos entre sí y cómo se reparten las responsabilidades. Ayudan a que el flujo de comunicación y las acciones entre partes del programa sean más claras y flexibles. Ejemplos típicos son el Observer (que notifica automáticamente a varios objetos cuando algo cambia), el Strategy (que permite cambiar el comportamiento en tiempo de ejecución) o el Command (que convierte una acción en un objeto que se puede ejecutar después).
